What to Submit

  • Build a small app or game during the in-person coding exam (1–4 hours, on your own laptop).
  • Push code to a remote repository with granular git commits throughout the exam.
  • Include a README explaining the project structure and how to run it.
  • Submit code link via the Learning Platform Hand-ins section.
  • Book your exam slot in advance.

What Gets You a Level 3 (Pass)

Pass/fail module. Pass requires meeting ALL qualification objectives:

  • Understand and use basic programming language features (functions, loops, conditions, arrays).
  • Use the command-line interface (CLI).
  • Apply basic version control (git) with granular commits showing progression.
  • Use an integrated development environment (IDE).
  • Understand iterative development.
  • Debug and troubleshoot computer programs.

Hard Rules

NO generative AI during the exam or for the hand-in. This module covers fundamentals that must be your own work.
Granular git commits required. A significant amount of code added in a single large commit = very high likelihood of failing.
OS_01 (Introduction to Software Engineering) is a prerequisite. Confirm it is completed before booking.
Bring your own laptop with IDE, git, and chosen language pre-installed. No setup time during exam.
